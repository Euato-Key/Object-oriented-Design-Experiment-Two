# 物联网设备知识图谱交互管理平台 - 实验二报告导读

## 1. 报告简介
本文件夹包含了“物联网设备知识图谱交互管理平台”实验二的所有产出物。报告基于“4+1”视图模型进行系统架构设计，并对四个核心子系统进行了详细设计。

## 2. 文件结构说明

### 2.1 核心报告文档 (Markdown)
建议按照以下顺序阅读报告：

1.  **[01-概述与系统架构.md](./01-概述与系统架构.md)**
    *   **内容**：项目背景、目的、范围、定义及缩写、参考文档。
    *   **核心**：系统整体架构设计的“4+1”视图（场景、逻辑、开发、物理、进程）。
2.  **[02-子系统A详细设计.md](./02-子系统A详细设计.md)** (设备接入与安全认证)
    *   **内容**：类图、顺序图、状态图、活动图、接口设计、UI原型（注册、登录、列表）。
3.  **[03-子系统B详细设计.md](./03-子系统B详细设计.md)** (设备监控与安全运维)
    *   **内容**：类图、顺序图、状态图、活动图、接口设计、UI原型（仪表盘、告警处理）。
4.  **[04-子系统C详细设计.md](./04-子系统C详细设计.md)** (设备资产与配置管理)
    *   **内容**：类图、顺序图、状态图、活动图、接口设计、UI原型（部署向导）。
5.  **[05-子系统D详细设计.md](./05-子系统D详细设计.md)** (知识图谱分析与智能服务)
    *   **内容**：类图、顺序图、状态图、活动图、接口设计、UI原型（图谱分析、分析报告）。
6.  **[06-数据库设计.md](./06-数据库设计.md)**
    *   **内容**：完整ER图、MySQL数据字典（12张表）、图数据库模型。
7.  **[07-用户界面设计.md](./07-用户界面设计.md)**
    *   **内容**：UI设计理念、所有8个界面原型的汇总说明。
8.  **[08-非功能性需求与总结.md](./08-非功能性需求与总结.md)**
    *   **内容**：可靠性、性能、可维护性等非功能性需求设计。
9.  **[09-实现模型与风险汇总.md](./09-实现模型与风险汇总.md)**
    *   **内容**：技术选型、实现策略、风险分析、用例覆盖总结、修订历史。

### 2.2 资源文件夹
*   **`ui/`**: 包含8个HTML界面原型文件，可直接在浏览器打开预览。
    *   `login.html`: 登录页
    *   `dashboard.html`: 仪表盘
    *   `device_list.html`: 设备列表
    *   `topology.html`: 图谱分析
    *   `device_register.html`: 设备注册 (新增)
    *   `alert_handle.html`: 告警处理 (新增)
    *   `deploy_wizard.html`: 部署向导 (新增)
    *   `analysis_report.html`: 分析报告 (新增)
*   **`diagrams/`**: 包含所有Mermaid格式的UML图表源码 (`.mmd`)。

## 3. 如何查看
*   **报告文档**：使用支持Markdown的编辑器（如Typora, VS Code）打开 `.md` 文件，即可预览内嵌的图表。
*   **界面原型**：双击 `ui/` 文件夹下的 `.html` 文件，使用Chrome或Edge浏览器查看交互效果。
*   **图表源码**：图表已嵌入文档，如需修改，可编辑 `diagrams/` 下的 `.mmd` 文件。
