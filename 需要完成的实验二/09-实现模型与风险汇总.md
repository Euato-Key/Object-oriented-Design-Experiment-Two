# 物联网设备知识图谱交互管理平台 - 实验二报告

## 09. 实现模型与风险汇总

### 1. 实现模型设计

#### 1.1 技术选型与开发环境
- **后端框架**：Spring Boot 3.x (Java 17)
- **前端框架**：Vue.js 3 + Element Plus
- **数据库**：
  - MySQL 8.0 (关系型数据)
  - Neo4j 5.x (图数据)
  - InfluxDB 2.x (时序数据)
- **中间件**：RabbitMQ (消息队列), Redis (缓存)
- **开发工具**：IntelliJ IDEA, VS Code, Maven, Git

#### 1.2 模块实现策略
系统采用Maven多模块项目结构进行管理：
- `iot-platform-common`: 公共工具类、DTO定义
- `iot-platform-access`: 子系统A实现 (Netty Server)
- `iot-platform-monitor`: 子系统B实现 (Stream Processing)
- `iot-platform-asset`: 子系统C实现 (Web API)
- `iot-platform-graph`: 子系统D实现 (Graph Algorithms)

### 2. 设计风险与未鉴别因素分析

#### 2.1 潜在风险
1. **图数据库性能风险**：随着设备节点数量达到百万级，复杂图查询（如多跳路径查找）可能出现延迟。
   - *缓解措施*：引入图计算引擎（如Spark GraphX）进行离线预计算。
2. **数据一致性风险**：MySQL与Neo4j之间的数据同步可能存在延迟或不一致。
   - *缓解措施*：使用Canal监听MySQL Binlog实现准实时同步，并增加定期对账机制。
3. **设备接入安全风险**：私有协议设备接入可能存在漏洞。
   - *缓解措施*：强制要求通过边缘网关进行协议转换和统一认证。

#### 2.2 未鉴别因素
- 具体部署环境的硬件配置（CPU/内存）尚未最终确定，需根据压测结果调整。
- 第三方AI模型的具体版本和授权模式待确认。
